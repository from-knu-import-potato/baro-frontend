A 제안 배경 및 목적

BARO가 탄생한 이유와 해결하고자 하는 문제, 기획 목적을 정의한다.

구상 배경 — 우리가 발견한 문제

국내 소규모 외식업체는 약 60만 개 이상(통계청 기준)에 달하지만, 이 중 대다수는 여전히 재고 관리를 수기 장부나 메모앱으로, 발주는 사장님의 경험과 직감에 의존하여 진행하고 있다. 기존 POS 시스템은 결제 처리 중심으로 설계되어 있어 재고 관리와의 연계가 없거나 매우 제한적이며, 도입 비용도 월 수십만 원에 이르러 소규모 가게에게는 부담이 크다. 배달의민족, 쿠팡이츠 같은 외부 플랫폼은 주문을 연결해주지만 가게의 재고나 운영 데이터와는 단절되어 있다.

이러한 환경에서 소규모 가게 사장님들은 매일 같은 수동 작업을 반복하고 있다. 냉장고를 직접 열어 재고를 눈으로 확인하고, 거래명세서를 수기로 옮겨 적고, 홀에서 주문을 받으면서 동시에 음료를 제조한다. 이 과정에서 발주를 너무 일찍 하거나 놓치는 일이 생기고, 유통기한 지난 재료를 뒤늦게 발견해 폐기하는 손실이 반복된다.

문제 정의

#문제원인BARO의 해결 방식
1수기 재고관리의 비효율디지털 입력 도구 부재, 입력 번거로움OCR 자동 파싱으로 촬영 한 번에 입고 처리
2발주 타이밍 파악의 어려움데이터 기반 분석 없이 직감에 의존AI 발주 가이드가 소비 데이터 기반으로 적정 발주량·시점 추천
3주문 접수 인력 필요홀 서빙과 주문 접수를 동시에 처리테이블 QR 비대면 주문으로 손님이 직접 주문
4판매와 재고 데이터의 단절주문 → 재고 차감이 자동으로 연결되지 않음마감하기 기능으로 오늘 판매된 메뉴 기반 재고 자동 차감
5메뉴판 제작·수정 비용메뉴 변경 시 인쇄물 재제작 필요디지털 QR 메뉴판으로 실시간 메뉴 수정 및 3가지 뷰 커스텀 제공

기획 목적

  • 운영 자동화: 반복적이고 시간이 많이 걸리는 재고 입력, 발주 판단, 마감 계산을 자동화하여 사장님이 고객 서비스와 요리에 집중할 수 있도록 한다.
  • 데이터 기반 운영: 경험과 직감에 의존하던 운영 방식을 데이터 기반 의사결정으로 전환하여 식재료 낭비와 손실을 줄인다.
  • 올인원 연결: 주문 → 재고 → 발주 → 마감이 하나의 플랫폼에서 유기적으로 연결되도록 하여 데이터 단절로 인한 업무 중복을 제거한다.
  • 진입 장벽 최소화: 복잡한 ERP나 POS 없이도 스마트폰 수준에서 바로 사용 가능한 낮은 기술 장벽의 서비스를 제공한다.

1 서비스 개요

BARO가 무엇을 하는 서비스인지 한 단락으로 정의한다.

BARO(바로)는 소규모 카페·식당 사장님을 위한 OCR·AI 기반 통합 가게 운영 SaaS이다.

손님의 주문 접수부터 재고 관리, AI 발주 추천, 하루 마감 정산까지 가게 운영에 필요한 핵심 업무를 하나의 플랫폼에서 처리할 수 있도록 한다. 특히 거래명세서 촬영 한 번으로 입고 데이터를 자동 등록하는 OCR 기능과, 테이블 QR을 통한 비대면 주문 시스템이 핵심 차별점이다.

2 핵심 가치 제안

BARO가 사장님에게 제공하는 5가지 핵심 가치.

#가치설명
1주문 자동화테이블 QR → 손님 주문 → 사장님 실시간 수신
2입고 자동화거래명세서 촬영 → OCR 파싱 → 재고 자동 등록
3발주 가이드재고 데이터 기반 AI 발주 추천 및 이유 제공
4마감 자동화판매 메뉴 + 레시피 기반 재고 자동 차감
5통합 대시보드주문·재고·매출 현황을 한 화면에서 파악

3 사용자 정의

서비스와 상호작용하는 두 가지 사용자 유형.

구분설명
사장님 (Owner)서비스의 주 사용자. 가게를 운영하며 주문 수락, 재고 관리, 발주, 마감 등을 수행한다.
손님 (Guest)QR을 통해 주문 페이지에 접근하는 비회원 사용자. 별도 로그인 없이 주문만 가능하다.

B 사용자(타겟층) 분석 및 정의

BARO의 핵심 타겟 사용자를 페르소나 기반으로 상세 정의한다.

주요 타겟 — 사장님 (Owner)

BARO의 핵심 사용자는 1~5인 규모의 소규모 카페·식당을 운영하는 사장님이다. 연령대는 30~50대이며, 스마트폰 사용에는 익숙하지만 복잡한 소프트웨어나 ERP 도구에는 진입 장벽을 느끼는 경우가 많다. 매일 10~14시간의 영업을 혼자 혹은 최소 인력으로 운영하면서 주문 접수, 재고 파악, 발주, 정산까지 모든 업무를 도맡아 처리한다.

핵심 Pain Point

보조 타겟 — 손님 (Guest)

손님은 서비스의 직접적인 구매 고객이 아니라, QR 주문 기능을 통해 서비스와 접점을 갖는 비회원 간접 사용자이다. 별도 앱 설치나 로그인 없이 테이블의 QR을 스캔하기만 하면 메뉴판을 보고 주문할 수 있어야 하며, 손님 입장에서는 기존 종이 메뉴판을 보고 주문하는 것과 다름없는 간단한 경험을 제공해야 한다.

4 기능 정의

BARO를 구성하는 9개 기능 도메인의 상세 정의.

기능 개요

BARO는 가게 운영의 처음부터 끝까지를 커버하는 9개의 핵심 기능으로 구성된다.

회원/인증은 카카오 소셜 로그인 하나로 간편하게 가입하고 로그인할 수 있도록 제공한다. 가게 데이터가 남아있는 동안에는 탈퇴가 제한되어 데이터 무결성을 보호한다.

가게 초기 세팅은 사장님이 처음 서비스에 접속했을 때 가게 운영에 필요한 기반 데이터를 구성하는 단계이다. 재고 수량은 입력하지 않으며, 어떤 재료가 있는지 항목 구조만 잡아두는 것이 목적이다.

메인 대시보드는 사장님이 가게를 운영하는 내내 열어두는 중심 화면이다. 실시간 주문 현황, 이번 달 가게 요약, 매출 그래프, OCR 빠른 입고, 메모 등이 한 화면에 배치된다.

주문 시스템은 테이블마다 비치된 QR 코드를 손님이 스캔하면 메뉴판 페이지로 이동하는 방식이다. 주문은 실시간으로 사장님 대시보드에 전달된다.

전체 재고 현황은 가게 보유 재고를 단일 페이지 리스트로 제공한다. 상태 기준 필터, 즐겨찾기, 정렬, 검색 기능을 포함한다.

OCR 기반 재고 입고 처리는 거래명세서 이미지를 업로드하면 OCR이 자동으로 파싱하여 입고 등록 화면을 구성하는 기능이다. 미등록 식자재는 해당 화면에서 즉시 등록 가능하다.

발주 가이드는 재고 데이터 기반으로 AI가 발주 필요 식자재를 분석하여 권장 발주량, 긴급도, 서술형 추천 이유를 제공한다.

마감하기는 당일 판매된 메뉴와 레시피를 기반으로 식자재 이론 소비량을 자동 계산하고, 사용자가 검토·수정 후 재고를 차감하는 기능이다.

가게 설정은 메뉴, 레시피, 식자재, 테이블, 메뉴판 뷰 등 가게 정보를 언제든 수정할 수 있는 관리 페이지이다.

4-1 회원 / 인증

카카오 소셜 로그인을 기반으로 한 간편 인증 시스템을 제공한다.

항목내용
로그인 방식카카오 OAuth 소셜 로그인 단일 제공
닉네임카카오에서 닉네임을 받아오며, 회원 설정 페이지에서 수정 가능
회원 탈퇴가게 데이터(재고, 메뉴, 레시피 등)가 남아있는 경우 탈퇴 불가
탈퇴 절차가게 데이터를 먼저 초기화(삭제) 한 이후에만 탈퇴 가능
토큰 관리JWT Bearer Token, refresh token 기반 자동 갱신

4-2 가게 초기 세팅

사장님이 처음 서비스에 가입한 후 가게 운영에 필요한 기본 데이터를 등록하는 단계이다. 이 단계에서는 재고 수량을 등록하지 않으며, 가게 구조(메뉴, 레시피, 식자재 항목)만 설정한다.

Step 1. 가게 정보 입력

필드설명
가게 이름사용자가 직접 입력
카테고리예: 카페, 한식, 양식, 분식 등 선택
업종 타입예: 매장형, 포장 전문, 배달 전문 등 선택

Step 2. 메뉴 등록

필드설명
메뉴명가게에서 판매하는 메뉴 이름
가격판매 단가
메뉴 이미지(선택) 메뉴 사진 업로드
설명(선택) 메뉴 소개 문구

Step 3. 식자재 등록

⚠️ 식자재 항목(마스터 데이터)을 등록하는 단계이며, 재고 수량·유통기한은 이 단계에서 입력하지 않는다.
필드설명
식자재 이름예: 감자, 우유, 에스프레소 원두
단위g, ml, 개 중 선택
최소 재고 기준 (안전 재고)발주 가이드 및 재고 부족 알림의 기준이 되는 최솟값

Step 4. 레시피 등록

필드설명
연결 메뉴어느 메뉴의 레시피인지 선택
사용 식자재등록된 식자재 중 선택
사용량해당 메뉴 1인분 기준 사용량 (단위 포함)

핵심 원칙

  • 초기 세팅에서는 재고 수량을 등록하지 않는다. (항목 구조만 등록)
  • 실제 보유 재고 수량·유통기한의 최초 등록은 대시보드 최초 진입 시 모달을 통해 진행한다.

4-3 메인 대시보드

사장님이 가게를 운영하는 동안 항상 열어두는 운영 컨트롤 타워 화면이다.

4-6 OCR 기반 재고 입고 처리

거래명세서를 촬영하거나 이미지 업로드하면 OCR이 재고 데이터를 파싱하여 입고 처리를 자동화한다.

입고 처리 흐름

이미지 업로드 또는 촬영 → OCR 파싱 (서버 처리) → 파싱 결과 UI 표시 → 사용자 검토 및 수정 → 미등록 식자재 즉시 등록 (필요 시) → 입고 확정 → 재고 현황 반영

단위 환산 규칙

OCR 인식 단위변환 처리
kgg으로 환산 (× 1,000) 후 서버 전송
Lml으로 환산 (× 1,000) 후 서버 전송
g, ml, 개그대로 전송
단위 환산은 프론트엔드에서 처리 후 서버에 표준 단위(g, ml, 개)로 전송한다.
OCR 결과는 자동으로 확정되지 않으며, 반드시 사용자의 검토 및 수동 확정 단계를 거친다.

D 아이디어의 창의성 및 구현 가능성 분석

BARO의 차별화 포인트와 기술적 실현 가능성을 분석한다.

창의성

① 주문 → 재고 → 마감을 잇는 데이터 사이클 완성

기존 소규모 외식업 솔루션은 POS(주문/결제)와 재고 관리가 서로 분리되어 있다. 사장님은 주문을 POS로 받고, 재고는 별도로 수기 관리하며, 마감은 다시 수동으로 계산하는 3중 구조다. BARO는 손님 주문이 들어오면 메뉴 레시피를 통해 자동으로 식자재 소비를 추적하고, 마감하기 기능으로 하루 사용량을 재고에서 차감함으로써 이 세 가지를 하나의 흐름으로 연결한다.

② 비대면 QR 주문 + 디지털 메뉴판 일체화

QR 주문 시스템은 별도의 앱 설치 없이 테이블 번호가 연동된 URL로 동작한다. 사장님은 메뉴판을 별도로 제작하지 않아도 되고, 3가지 뷰 타입(카드형/리스트형/심플형)으로 메뉴판 스타일을 직접 커스텀할 수 있다. 메뉴를 바꾸면 즉시 손님 화면에도 반영되어 인쇄 비용이 발생하지 않는다.

③ OCR 미등록 식자재 즉시 등록 흐름

기존 OCR 솔루션은 미등록 항목을 오류로 처리하거나 별도 화면으로 이탈시킨다. BARO는 OCR 결과 화면 안에서 미등록 식자재를 즉시 등록하고 바로 입고 처리를 이어갈 수 있도록 설계했다. 업무 흐름이 끊기지 않고 한 화면에서 완결된다.

④ 서술형 AI 발주 이유 제공

대부분의 발주 추천 기능은 "이만큼 발주하세요"라는 수량만 제시한다. BARO의 AI 발주 가이드는 "왜 지금 이 재료를 발주해야 하는가"의 분석 근거를 자연어로 함께 제공한다. 사장님이 AI 추천을 납득한 뒤 발주 결정을 내릴 수 있도록 한다.

E 서비스를 통한 사회/경제적 가치창출 효과

BARO가 사회와 경제에 미치는 기대 효과 및 파급력을 분석한다.

사회적 파급효과

① 소상공인 디지털 전환 지원

국내 소규모 외식업체 약 60만 개 중 디지털 솔루션을 도입한 곳은 극히 일부에 불과하다. BARO는 복잡한 ERP나 고가의 POS 없이도 스마트폰으로 바로 접근 가능한 낮은 진입 장벽의 디지털 운영 도구를 제공한다. 이는 소규모 가게의 디지털 전환을 실질적으로 지원하며, 기술 소외 계층인 소상공인의 디지털 격차를 좁히는 데 기여한다.

② 식품 폐기 감소 및 환경 기여

정확한 재고 관리와 AI 발주 가이드를 통해 과잉 구매로 인한 식재료 낭비를 줄인다. 유통기한 임박 알림 기능은 사장님이 폐기 직전 재료를 먼저 소진하도록 유도하여 식품 낭비를 능동적으로 방지한다. 이는 UN SDG(지속가능발전목표) 12번 '책임감 있는 소비와 생산'과도 연결된다.

③ 소규모 가게의 1인 운영 가능성 확대

QR 비대면 주문 시스템 도입으로 홀 서빙 중 주문을 받는 인력이 없어도 주문 접수가 가능해진다. 이는 최소 인력으로 운영해야 하는 소규모 가게의 인력 부담을 줄이고, 1인 창업·운영의 실현 가능성을 높인다.

④ 지역 소상공인 경쟁력 강화

대형 프랜차이즈가 독점하던 디지털 운영 효율성을 소규모 독립 가게도 갖출 수 있게 함으로써 지역 경제의 다양성을 보호하고 소상공인의 생존율을 높이는 데 간접적으로 기여한다.

경제적 파급효과

① 과잉 발주 비용 절감

AI 발주 가이드를 통해 필요한 양만큼만 적시에 발주할 수 있게 되면, 과잉 재고로 인한 폐기 손실이 줄어든다. 소규모 카페 기준 월 평균 식재료 비용이 100~300만 원이라고 할 때, 과잉 발주 및 폐기를 10~20% 줄이면 월 10~60만 원의 비용 절감 효과가 발생한다.

② 운영 시간 단축에 따른 기회비용 절감

  • OCR 자동 입고: 거래명세서 수기 입력 시간 → 촬영 한 번으로 대체 (하루 약 10~20분 절약)
  • QR 주문: 홀 주문 접수 이탈 시간 제거 (피크 타임 실수 감소)
  • 마감 자동화: 수기·엑셀 정산 30분 → 마감하기 기능으로 5분 이내 처리
  • 하루 약 1~2시간 절약 시 월 30~60시간의 운영 시간을 서비스 품질 향상에 재투자 가능

③ 매출 데이터 기반 메뉴 전략 최적화

대시보드의 매출·소비 그래프를 통해 잘 팔리는 메뉴와 식재료 소비 패턴을 데이터로 파악할 수 있다. 이를 바탕으로 저마진 메뉴를 줄이거나 고회전 식재료에 집중 투자하는 등 데이터 기반 메뉴 전략이 가능해진다.
